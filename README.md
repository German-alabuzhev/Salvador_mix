# Чат-бот ВКонтакте для подбора кальянных миксов «Salvador Mix»

Интерактивный чат-бот для социальной сети ВКонтакте. Бот собирает предпочтения пользователя (желаемую крепость, направление вкуса, конкретный профиль) и на основе этих данных генерирует случайный сбалансированный микс из трёх компонентов, рассчитывая пропорции в зависимости от требований к охлаждающему эффекту.

---

## 📁 Структура проекта и предназначение файлов

Проект спроектирован с разделением на логические слои (бизнес-логика, интерфейс, работа с данными), что исключает загромождение главного модуля.

* **main.py** — Центральный исполняемый файл приложения. Отвечает за инициализацию сессии VK API, запуск LongPoll-слушателя событий, маршрутизацию шагов пользователя (конечный автомат состояний) и синхронизацию кэша при старте.
* **config.py** — Файл конфигурации. Хранит секретный токен доступа (`TOKEN`) для авторизации на серверах ВКонтакте.
* **.gitignore** — Технический файл, указывающий системе Git игнорировать виртуальное окружение (`.venv`), конфиг с токенами и локальные файлы БД при отправке в репозиторий.
* **database/** — Пакет для управления локальным хранилищем данных.
  * **database.py** — Модуль для работы с СУБД SQLite3. Содержит логику создания таблиц (`init_db`), очистки и перезаписи кэша (`update_tobaccos_cache`), а также выполнения оптимальных SQL-запросов для фильтрации.
  * **hookah.db** — Бинарный файл локальной базы данных SQLite. Создаётся автоматически при первом запуске приложения.
* **keyboards/** — Пакет управления визуальным интерфейсом бота.
  * **keyboards.py** — Содержит структуры и конфигурации интерактивных клавиатур (кнопок) для каждого отдельного шага опроса.
* **services/** — Пакет, содержащий сервисы бизнес-логики.
  * **data_provider.py** — Модуль сетевого взаимодействия. Отвечает за обращение к удаленному источнику (Google Таблицы), скачивание актуального CSV-файла и его первичную обработку.
  * **mix_generator.py** — Модуль логики миксования. Получает отфильтрованный из БД список табаков, выбирает случайную тройку и динамически высчитывает процентное соотношение в зависимости от выбора холодка («Да»/«Нет»).

## 🗄️ Описание структуры локальной базы данных

Для реализации требований кэширования данных в проекте используется реляционная СУБД **SQLite3**. Информация о доступных табаках хранится в таблице **`tobaccos`**:

| Имя поля таблицы | Тип данных | Описание |
| :--- | :--- | :--- |
| **id** | INTEGER | Первичный ключ (`PRIMARY KEY`) с автоматическим инкрементом. |
| **name** | TEXT | Название бренда и конкретного вкуса табака (например, *Darkside Тропический микс*). |
| **strength** | TEXT | Текстовый критерий крепости продукта (*лёгкая*, *средняя*, *крепкая*). |
| **taste** | TEXT | Направление вкусовой палитры (*фруктовый*, *ягодный*). |
| **profile** | TEXT | Конкретный профиль вкуса (*сладкий*, *кислый*). |

**Реализация гибридного кэширования:** 
При запуске `main.py` программа выполняет сетевой запрос к Google Таблице через сервис `data_provider.py`. Если запрос успешен, модуль базы данных выполняет транзакцию: полностью очищает таблицу (`DELETE FROM tobaccos`) и производит пакетную вставку свежих строк. 

Если у сервера пропадает интернет-соединение, блок `try-except` отлавливает ошибку, бот не падает и бесперебойно продолжает работу, выполняя SQL-запросы `SELECT` к уже сохраненному локальному кэшу в `hookah.db`.
